# Getting Support

Thank you for using Core Provisioner! This document outlines the ways you can get help.

## Available Resources

### Documentation

- **[README](README.md)** — what the skeleton is, how consumers fetch it, and how `Hosts.yml`/`Hosts.rb` fit together
- **[examples/Hosts.yml](examples/Hosts.yml)** — a complete, commented example configuration
- **[RELEASE.md](RELEASE.md)** — how releases and versioned archives are produced

### Issues and Bug Reports

- **[GitHub Issues](https://github.com/STARTcloud/core_provisioner/issues)** — report bugs or request features
- Use the appropriate issue template and search existing issues first to avoid duplicates

## Support Expectations

Core Provisioner is maintained with limited development resources:

- **Response time**: usually a few days, but this varies with workload and complexity
- **Priority**: security issues and provisioning-breaking bugs receive the highest priority
- **Feature requests**: evaluated against community need and available time

## Getting Quality Help

1. **Use the issue templates** and fill in the environment details (host OS, Vagrant version, provider)
2. **Include the relevant `Hosts.yml` portion** (redact secrets)
3. **Include the actual vagrant/ansible output**, not a paraphrase
4. **Test with the latest release** when possible

## Contributing Back

- **Document solutions** you find to help others
- **Contribute bug fixes** when you solve issues
- **Improve documentation** based on your experience

See our [Contributing Guide](CONTRIBUTING.md) for more information.
